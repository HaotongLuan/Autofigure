AutoFigure Paper / Tools Notes

本页只保留 AutoFigure 相关工作、方法与 benchmark 表格，以及当前自动生图的典型 failure case。

1. AutoFigure 代表性工作脉络

  • 2026-02｜AutoFigure：提出 Review-and-Refine 自动科学插图生成流程，从长文本生成可编辑 SVG / mxGraph XML。
  • 2026-02｜FigureBench：随 AutoFigure 提出的长上下文科学插图 benchmark，包含 3,300 个文本-图表对。
  • 2026-03｜AutoFigure-Edit：面向已有科学图的局部编辑，结合语义分割与自然语言指令修改 SVG 元素。
  • 2026｜DeepScientist + AutoFigure：将 AutoFigure 作为自动科研系统中的图表生成模块，用于论文写作阶段的插图落盘。

4. Failure Case 与具体例子

以下是实际运行 AutoFigure pipeline 复现的 failure case，含完整输入/输出/评估：Failure Case 复现详情

Failure Case 1：长文本输入导致关键模块遗漏

问题：当输入包含完整论文方法段落时，模型容易抓住主干流程，但遗漏辅助模块、反馈路径或损失函数。

输入：
请根据论文方法部分生成框架图：系统包含 Encoder、Retriever、Planner、Renderer、Critic。Critic 的反馈会回传给 Planner 和 Renderer；此外还有一个 Memory Bank 存储历史图例，用于辅助 Retriever。

可能输出：
Encoder → Retriever → Planner → Renderer → Critic

错误点：
遗漏 Memory Bank；Critic 只接在末尾，没有画出回传给 Planner / Renderer 的反馈边。

Failure Case 2：空间拓扑和箭头语义错误

问题：生成图看起来完整，但箭头方向、模块层级或连接关系与文本描述不一致。

输入：
画一个三阶段流程图：Data Preprocessing 输出给 Model Training；Model Training 输出给 Evaluation；Evaluation 的诊断结果只反馈给 Data Preprocessing，不直接反馈给 Model Training。

可能输出：
Data Preprocessing → Model Training → Evaluation
                 ↑_______________________|

错误点：
反馈箭头指向了 Model Training，和文本要求的“只反馈给 Data Preprocessing”不一致。
