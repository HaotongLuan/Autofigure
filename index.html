5. Failure Cases 与现存问题

5.1 结构性幻觉 (所有系统共性问题)

表现：箭头指向错误的模块；连接线穿越而非绕过障碍物；数据流方向反向。

原因：当前 VLM 对空间拓扑关系的理解仍是概率性的, 不具备真正的几何推理能力。符号中间态只缓解不根治。

证据：SciFlow-Bench 的逆解析实验发现, 即使是最优模型在 "箭头正确性" 维度上通过率仅约 60%。

5.2 文本渲染乱码

表现：扩散模型生成的图中的文字出现拼写错误、乱码字符、截断、或字体大小不统一。

原因：扩散过程在潜空间操作, 文字这种离散符号在连续潜空间中缺乏锚定。OCR 后修正只能缓解。

证据：PosterVerse 实验显示 GPT-4o 和 Gemini 2.0 在直接生成含中文文字的图片时, 字符识别率 (CR) 低于 60%。SridBench 将 "文字信息完整性" 列为六大维度之首。

5.3 科学公理违背

表现：生成的分子结构违反化合价；电路图中出现短路；受力分析遗漏关键力。

原因：模型学的是视觉统计分布, 而非领域公理。当前没有任何系统内嵌了化学/物理定律验证。

证据：SciGenBench 专门引入"科学合理性"维度, 发现现有 T2I 模型在该维度得分远低于布局和美学维度。

5.4 长上下文信息遗漏

表现：从完整论文生成架构图时, 遗漏关键模块、丢失次要但重要的连接。

原因：即使有符号中间态, LLM 的注意力在超长上下文中仍会稀释。AutoFigure 对论文类内容的胜率仅 53%。

证据：FigureBench 数据显示, 输入文本越长, AutoFigure 的 Critic 评分成递降趋势。

5.5 跨领域泛化失败

表现：在 CS/ML 论文上表现良好的系统, 面对化学分子式、生物通路图、工程原理图时几乎不可用。

原因：几乎所有基准 (PaperBananaBench, FigureBench) 均以 CS/ML 论文为主, 训练数据同质化严重。

证据：SridBench 覆盖 13 个学科, 同模型在不同学科间得分方差极大。尚无公开系统在生命科学领域达到可用水平。

8. 可改进方向与未来研究机会

8.1 内嵌领域知识约束 (高优先级)

现状：所有系统均依赖通用 VLM, 不具备化学/生物/物理领域公理。

方向：在符号中间态审查阶段引入领域规则引擎——例如生成分子图时强制检查化合价、生成电路图时检查 KCL/KVL。可将 SciGenBench 的科学合理性维度作为训练信号。

8.2 多轮交互式编辑能力 (当前缺口大)

现状：除 APEX 外, 大部分系统是"一次性生成"。实际科研中, 图表需经数轮导师/审稿人反馈修改。

方向：借鉴 APEX 的 Review-and-Adjustment 机制, 但扩展到 自然语言指令的局部修改——如"把第三个箭头从蓝色改成红色"、"交换模块 A 和 B 的位置"——而非重新生成整张图。

8.3 跨学科迁移学习

现状：SridBench 显示, 现有系统在不同学科间性能方差极大。尚无针对生命科学/化学/工程的有效方案。

方向：收集多学科科学图表数据集并进行领域适配微调; 或利用 LLM 的跨学科知识迁移, 在符号中间态层面做领域映射。

8.4 动态科学动画的标准化评估

现状：LottieGPT/OmniLottie 能生成动画, 但没有任何基准评估"科学动画"的质量——时序逻辑、因果传递、动画节奏是否传达正确的科学含义。

方向：建立 SciAnimationBench: 收集算法可视化、生物过程动画、物理模拟动画, 设计时间维度评估协议。

8.5 端到端科研工作流集成

现状：AutoFigure 集成到了 DeepScientist, PaperBanana 配对 ScholarPeer。但这些都是独立系统, 互不兼容。

方向：建立统一的 AutoSciFigure API 标准, 使任何 AI Scientist 系统可即插即用地调用图表生成模块。类似 MCP 协议, 但专门为科学图表设计。

8.6 文本渲染的根本性解决

现状：扩散模型生成文字仍然不可靠。PosterVerse 的 HTML 方案和 AutoFigure 的 SVG 方案是目前唯二可靠的路径。

方向：将 HTML/SVG 排版引擎与 VLMs 深度耦合, 让所有文本标签在渲染后处理阶段由排版引擎原生生成, 而非依赖扩散像素。

8.7 结构化矢量表示的通用 token 化方案

现状：LottieGPT/HiVG/VecGlypher 各有自己的 token 化策略, 互不兼容。

方向：研究通用的科学矢量图 token 化标准——覆盖 SVG、Lottie、TikZ、mxGraph——使单一模型能输出多种矢量格式。
